# 面向开源的 Codex

开源维护者常常在幕后做着关键工作，维持软件生态的健康。在过去一年中，Codex Open Source Fund（100 万美元）支持了需要 API credits 的项目，其中包括使用 Codex 驱动 GitHub pull request workflows 的团队。OpenAI 感谢这些维护者持续推动这项工作。

该基金现在为符合条件的维护者提供支持：向拥有写入权限的核心维护者提供为期六个月、包含 Codex 的 ChatGPT Pro，以及有条件访问 Codex Security 的资格。开发者应该使用自己偏好的工具编写代码，无论是 Codex、[OpenCode](https://github.com/anomalyco/opencode)、[Cline](https://github.com/cline/cline)、[pi](https://github.com/badlogic/pi-mono/tree/main/packages/coding-agent)、[OpenClaw](https://github.com/openclaw/openclaw)，还是其他工具；这个项目正是为了支持这类工作。

## 该项目包含什么

- 为期六个月、包含 Codex 的 ChatGPT Pro，用于日常编码、triage、review 和维护者 workflows
- 对需要更深入安全覆盖的 repositories，提供有条件的 Codex Security 访问
- 通过 Codex Open Source Fund 提供 API credits，用于在 pull request review、维护者自动化、release workflows 或其他核心 OSS 工作中使用 Codex 的项目

考虑到 GPT-5.4 的能力，团队会逐案审核 Codex Security 访问权限，以确保这些 workflows 获得所需的谨慎与尽责处理。

如果你是核心维护者，或运营一个广泛使用的公共项目，请申请。如果你的项目不完全符合标准，但在生态系统中扮演重要角色，也请申请并说明原因。

提交申请即表示你同意 [Codex for Open Source Program Terms](https://developers.openai.com/codex/codex-for-oss-terms)。
